# 0006 — Server functions are machine-checked against the real database layout

- **Date:** 12/07/2026
- **Status:** Accepted

## Decision

Every Edge Function — the server-side code that creates bookings, moves refund money, and sends notifications — is typed against a generated description of the actual database schema (the `<Database>` generic), via a managed copy at `supabase/functions/_shared/database.types.ts`. A CI check fails the build if that copy drifts out of sync.

## Why

Before this, a typo'd column name or a wrongly-shaped write in an Edge Function would compile fine and only fail (or worse, silently misbehave) in production — on exactly the code paths that touch money. With the generated types in place, those mistakes are caught on the developer's machine or in CI before anything deploys. The copy is generated by a script rather than maintained by hand, and the CI gate means it can't quietly go stale. Shipped as PR #415 (12/07/2026).
